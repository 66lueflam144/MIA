DLL Hijacking

DLL 劫持是一种技术，它通过欺骗第三方代码加载错误的库 （DLL） 来将第三方代码注入合法进程 （EXE） 中。

  • 最常见方法是将外观相似的 DLL 放在搜索顺序中高于预期 DLL 的位置，从而使 Windows 库加载程序首先选择您的 DLL。

缺点

DLL Hijacking在加载->执行第三方代码上有一个缺点——加载器锁(Loader Lock).

  • Loader Lock对第三方代码有着严格的限制：创建进程(creating process)、执行网络I/O(doing network I/O)、调用注册表函数(calling registry functions)、创建图形窗口(creationg graphical)…
  • 尝试在其下执行任何操作都可能导致崩溃或者程序挂起。

DLL Hijacking ways

  • 1.使用Virtualprotect更改内存保护
    • 反恶意软件解决方案会标记VirtualProtect操作，特别是creating read-write-execute memory或者converting from read-write to read-execute这类操作。
    • 启用了任意代码保护 （ACG） 的进程会完全阻止创建或修改可执行内存
  • 2.修改指针
    • 例如，在Locker Lock释放后，存在于栈上的函数的返回地址会被修改以执行代码。（For instance, a function’s return address on the stack being modified to get code execution after Locker Lock is released.）
    • 但是指针是下一代exploit mitigations的目标，例如：即将推出的称为英特尔控制流强制技术 （CET） 的漏洞利用缓解措施中断。
