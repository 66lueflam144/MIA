CSRF

跨站请求伪造，也称One click attack，XSRF

  • 攻击者盗用用户身份以受害者名义进行发送请求（恶意的）。
  • 互联网上的许多站点仍对此毫无防备，以至于安全业界称CSRF为“沉睡的巨人”

原理

虽然很想搞个图片但发现太丑不如等我有空自己画

  • ROLE
    • User Browser
    • Trusted Web A
    • Hacked Web B
  • STEPS
      1. UB浏览并登入Web A
      1. Web A返回Cookie，该Cookie储存在UB的电脑上。
      1. User没有登出A网站，又去访问了Web B
      1. B发出访问A的请求，User轻轻一点
      1. User电脑根据请求去访问A
      1. Web A根据Cookie以为是验证过的用户来了，所以以为是User在进行操作。
  • 2 parts
    • 存储在本机的cookie
    • 不登出A的情况下访问B
      比较有趣的部分：
  • 1.登录一个网站后打开另一个tab访问其他网站
  • 2。cookie的有效期——会话何时结束（关闭浏览器并不意味着会话结束）
  • 3.Web B可能只是一个存在漏洞的网站（大多数情况下不是hacker自建（真有这么闲不如转行做web开发））

根据我的大致理解，这一个demo是被修改了post请求。

//before you transfer:

    const data = new URLSearchParams();
    for (const pair of new FormData(formRef.current)) {
      // pair[0] is the field name, pair[1] is the field value
      data.append(pair[0], pair[1]);
    }

    fetch('/transfer', {
      method: 'post',
      body: data,
    })

//after:

const body = new URLSearchParams('amount=10000&description=Gotcha!&to=Evil-Scammers');
fetch('https://dsb.victorzhou.com/transfer', {
  body,
  credentials: 'include', // include cookies, despite being a cross-origin request
  method: 'post',
  mode: 'no-cors',
});

所以可以看到const body的值被修改。
