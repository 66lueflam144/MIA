下载Module

有时候使用py的时候会报错说xx module没有，上网随便找一个下载命令发现好不靠谱，所以仔细找了一下实用的。

PIP安装

  • 1.先检测pip module是否存在
pip list

如果输出数据就表示存在该模块，否则就通过python -m ensurepip安装，之后pip install --upgrade pip更新一下（不搞也行，只是以防万一）

  • 2.pip install xxx即可下载。如果报错，大概是没有把pip路径添加到环境变量里面，所以更换python -m pip install xx。
  • 3.卸载就是pip uninstall xx
  • 4.timeout问题：增加参数由官网下载变为国内镜像下载：pip install xx -i URL --trusted-host xxx.com（xxx.com就是URL里面的网址）

多版本管理

起源是因为受不了没有cover的日子于是打算自己搞一个AI voice之类的东西，然后上GitHub找，然后在配置的过程中遇到python版本不对的情况，不想特复杂的设置，所以找到了pyenv这个工具。

install

官方Installation文档

  • 使用zip的进行下载（问就是其他的没这个省事，都不用修改outfile）
  • 解压到.pyenv这个名称的文件夹之下（自建的）
  • 进行环境变量修改（系统变量）：
    • 系统变量增加PYENV变量，value设置为xx\xx\pyenv-win
    • path分别增加%PYENV%\bin和%PYENV%\shims
  • 在cmd界面输入pyenv显示版本信息就安装成功了，

在第一次启用的时候好像（？）会报错：

请只运行你信任的脚本。虽然来自 Internet 的脚本会有一定的用处，但此脚本可能会损坏你的计算机。如果你信任此脚本，请使用 Unblock-File cmdlet 允许运行该脚本，而不显示此警告消息。是否要运行

根据Unblock-File进行修改就行了。

一个例子：

Unblock-File -Path C:\Users\User01\Documents\Downloads\PowerShellTips.chm

usage

常用：

pyenv installl --list #展示可安装python版本列表
pyenv install xxx #安装指定版本python
pyenv version #显示当前python版本
pyenv versions #显示已安装的python版本
pyenv global xxx #全局使用xxx版本python
pyenv shell xxx #为当前会话设置版本xxx，关闭就失效
pyenv local xxx #为当前目录设置版本xxx

不过一般不会用到后两个。
