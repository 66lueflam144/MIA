CORS

2023/10/30 LEARN

What is CORS

跨源资源共享 (CORS) 是一种浏览器机制，可以对位于给定域之外的资源进行受控访问。它扩展并增加了同源策略 ( SOP ) 的灵活性。

这种机制很有用，因为复杂的应用程序通常会在其客户端代码中引用第三方 API 和资源。
例如：

  • 应用程序可能使用浏览器从视频平台 API 提取视频，
  • 使用公共字体库中的字体，
  • 或显示来自国家天气数据库的天气数据。

CORS 允许客户端浏览器在传输任何数据之前向第三方服务器检查请求是否获得授权。

如果网站的 CORS 策略配置和实施不当，它也可能带来跨域攻击。CORS 不能防止跨源攻击，例如跨站点请求伪造( CSRF )。
为了防止CSRF，所有浏览器都在实施同源策略

同源策略

浏览器强制要求客户端只能向与客户端 URL 具有相同来源的资源发送请求。客户端 URL 的协议、端口和主机名都应与其请求的服务器相匹配。

原URL：http://xx.com/face2/dd.html

  • 来源相同（只有路径不同）：http://xx.com/face2/click/dd.html

  • 来源相同（只有路径不同）：http://xx.com/face/aa.html

  • 来源不同（协议不同）：https://xx.com/face2/dd.html

  • 来源不同（端口不同，默认为80）http://xx.com:81/face2/dd.html

同源策略非常安全，但却不够灵活

CORS工作原理

浏览器向应用程序服务器发送 HTTP 请求，以 HTTP 响应的形式接收数据，然后显示数据。在浏览器术语中，当前浏览器 URL 被称为当前来源，而第三方 URL 则是跨源的。

当您发出跨源请求时，请求-响应过程如下：

  • 浏览器在请求中添加来源标头，其中包含有关当前来源的协议、主机和端口的信息
  • 服务器检查当前来源标头，然后使用请求的数据和 Access-Control-Allow-Origin标头进行响应
  • 浏览器查看访问控制请求标头并与客户端应用程序共享返回的数据
  • 或者，如果服务器不想允许跨源访问，则会回复一则错误消息。
