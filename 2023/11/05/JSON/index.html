JSON&YAML&XML

2023/11/5 LEARN

what is JSON

JavaScript Object Notation (JS对象表示法)，即一种用于存储和传输数据的文本格式

  • 轻量级数据集交换格式
  • 用JS对象表示法编写的纯文本
  • 用于计算机之间发送数据
  • 与语言无关

why JSON

  • 如果学过JS，就能够轻松将JSON数据和JS对象进行互换。
  • 格式仅为文本，因此可以轻松发送且无编程语言限制。（不过现代很多编程语言也包含用于生成和解析JSON格式数据的代码）

XML

可扩展标记语言
XML用于描述结构化数据和序列化对象。

JSON vs XML

  • XML只能有一个值，并且每个属性在每个元素上最多可以出现一次。

  • XML将数据和元数据分开，而JSON没有。

  • JSON对象寻址时键到值的映射，而XML寻址发生在节点上。

  • XML标记名称不能包含任何字符&空字符&-，.或数字开头。JSON键可以。

  • XML 值是字符串，没有内置的类型安全性。XML 具有schema的概念，它允许强类型、用户定义类型、预定义标签和形式结构，从而允许对 XML 流进行形式验证。JSON 内置了多种类型，并且在JSON Schema中具有类似的模式概念。

  • XML 支持注释，而 JSON 不支持。

YAML

是一种人类可读的 数据序列化语言。它通常用于配置文件以及存储或传输数据的应用程序。

  • YAML1.2基于JSON语法。

  • YAML 具有 JSON 中不存在的许多附加功能，包括注释、可扩展数据类型、关系锚、不带引号的字符串以及保留键顺序的映射类型。

  • JSON序列化和反序列化比 YAML 快得多
