2.Basis

2.1 Web development

You should be familiar with the fundamentals of web development, such as HTML, CSS, JavaScript, and how web applications work. You should also know how to use web frameworks, such as Django, Flask, Ruby on Rails, etc. This will help you understand how web applications are built and how they can be vulnerable to attacks.

2.1.1 Fundamental of web development

  • HTML
  • CSS
  • JavaScript

这些均在W3school可以学到，不过还要结合漏洞才能知道需要重点注意什么代码。常常利用的是JS代码。

2.1.2 web applications work

大概为下面5个步骤：

    1. User Request（用户请求）: 用户通过打开web网页，触发发送一个请求给服务器。
    1. Web Server Processing（服务器加载）: 服务器接收到请求然后发送到相应的应用服务器，应用服务器再进行被请求的进程，比如向数据库查询值。
    1. Web Application Server Processing（应用服务器加载）: 应用服务器处理请求和运行所有后端逻辑。查询数据库、计算、或者任何某个请求所必要的任务的执行。
    1. Response Generation（构造响应）: 应用服务器完成上面的任务后，就会构造请求数据的结果。
    1. Response Delivery（发回响应）: 应用服务器把结果发送回服务器，服务器又发回客户端。

2.2 Networking

You should be familiar with the basics of networking, such as TCP/IP, HTTP, DNS, SSL/TLS, and how data is transmitted over the Internet. You should also know how to use networking tools, such as ping, traceroute, nmap, wireshark, etc. This will help you understand how web applications communicate and how they can be intercepted or manipulated by attackers.

2.2.2 networking tools

  • ping: 因特网包探索器，用于测试网络连接量的程序。Ping发送一个ICMP回声请求消息给目的地并报告是否收到所希望的ICMP回声应答。
  • ipconfig：查看ip掩码，网关，修改DNS servers，设置gateways多用于故障分析
  • net：几乎所有有关network的设置都可以使用net command
  • netstat: 网络统计，用于展示两台设备如何连接的细节。特别的可以展示个人网络连接的细节。click to learn more
  • traceroute
  • nmap

  • DevTools（开发者工具）

ICMP: the Internet Control Message Protocol, a network layer protocol used by network devices to diagnose network communication issues.
ICMP is part of the Internet protocol suite as defined in RFC 7922. It is used by network devices, including routers, to send error messages and operational information indicating success or failure when communicating with another IP address. For example, an error is indicated when a requested service is not available or that a host or router could not be reached.
ICMP differs from transport protocols such as TCP and UDP in that it is not typically used to exchange data between systems, nor is it regularly employed by end-user network applications (with the exception of some diagnostic tools like ping and traceroute).

2.3 Cryptography

You should be familiar with the basics of cryptography, such as encryption, decryption, hashing, digital signatures, and how they are used to protect data and ensure its integrity and authenticity. You should also know how to use cryptographic tools, such as openssl, pgp, bcrypt, etc. This will help you understand how web applications secure data and how they can be compromised by attackers.

2.4 Programming

You should be familiar with the basics of programming, such as variables, data types, control structures, functions, and how to write and debug code in one or more languages. You should also know how to use programming tools, such as IDEs, compilers, interpreters, debuggers, etc. This will help you understand how web applications function and how they can be exploited by attackers.
